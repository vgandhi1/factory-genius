Simulated multimodal edge evidence → MQTT → BM25 RAG (+ optional LLM) → a technician dashboard that returns preventive vs breakdown guidance with cited manual excerpts.

Python 3.11+ · FastAPI React 18 · Vite · Tailwind MQTT Mosquitto RAG BM25 status · reference prototype MIT
The problem

Sensors raise alarms. They don't tell you how to fix it.

$10k+/min

Unplanned stoppage cost on stamping presses, conveyors, and CNC lines.

Alarm ≠ action

Vibration / heat thresholds flag a fault but leave the technician guessing root cause and procedure.

Tribal knowledge

OEM manuals and fix history live in PDFs and senior heads — not at the machine, mid-incident.

Factory Genius closes the gap between detection and guided repair — human-in-the-loop, not autonomous control.

The concept

A copilot that fuses edge evidence with cited procedures

  • A Jetson-class edge fuses thermal (FLIR-class), optical, and directional-mic / spectrogram evidence.
  • On anomaly it ships a fused packet to a central brain over conveyance/+/anomaly.
  • The brain runs RAG over maintenance manuals so answers are actionable and cited, not generic.
  • This repo ships a fully runnable prototype of the guidance loop — simulated edge, real MQTT + RAG + dashboard.

Target brain (roadmap): industrially-tuned VLM + audio/spectrogram model on fused packets, orchestrated with RAG.

Edge evidence

Multimodal payloads, one fused packet

Thermal

FLIR-class °C + baseline. Drives critical-threshold and step-above-baseline heuristics.

Optical

RGB summary + image ref. Visual context for the central VLM (refs not echoed back).

Acoustic

STFT band-energy + crest factor. Bands map to bearings, belts, looseness, chatter.

audio_analysis.py — stft_band_energy_v1: a reference signal-processing path (not a certified classifier). Upload a WAV/FLAC clip → dominant band → RAG query hint.

The output technicians see

Preventive window vs breakdown response

Preventive / condition-based

Acoustic change with limited thermal rise · moderate thermal elevation (Δ 8–25°C).

→ Plan greasing, alignment verification, bearing assessment in a scheduled window before peak production.

Breakdown / immediate

Thermal ≥ 85°C critical threshold · large step (Δ ≥ 25°C) above baseline.

→ Stop and inspect per site policy before restart; treat as failure-in-progress.

Every diagnosis cites the retrieved manual excerpts (BM25) and appends LOTO + speed/load verification steps. LLM prompts isolate reference text in <data> blocks against injection.

Production path

From prototype to governed MLOps

RAG Retrieval

Now: BM25 on local Markdown. Next: embeddings + vector DB (Qdrant / Milvus / pgvector), hybrid sparse+dense.

LLM Gateway

Now: optional OpenAI-compatible client, template fallback. Next: private endpoint, PII redaction, prompt/version registry.

EVAL Quality

Now: manual dashboard review. Next: retrieval hit-rate / nDCG, golden sets per asset class, technician feedback loop.

OPS Serving

Now: uvicorn + static SPA, Docker Compose for MQTT. Next: per-service images, health checks, secrets via vault.

Roadmap

Toward real multimodal inference at fleet scale

  • On-device fusion on Jetson — FLIR-class thermal + RGB + directional-mic spectrograms, privacy band-pass, real firmware.
  • Central brain — industrially-tuned VLM + audio/spectrogram model on fused packets, orchestrated with RAG.
  • Vector search at scale — embeddings + Milvus / Qdrant / pgvector for the OEM manual corpus.
  • Coverage — stamping presses, CNC, conveyors, and shared plant services.
  • EAM connectors — SAP PM, Maximo + technician feedback loops.
